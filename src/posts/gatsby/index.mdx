# Gatsby

---

<img className="logo" src={Logo} alt="로고" />

**Gatsby**는 **React** + **JAM Stack**을 활용한 **정적 웹사이트 생성기**

먼저 Gatsby의 핵심인 JAM Stack에 대해 먼저 알아보겠습니다.

## JAM Stack?

공식 도큐먼트에 의하면 JAM Stack은 데이터 및 비즈니스 논리에서 웹 경험 계층을 분리하여 유연성, 확장성, 성능 및 유지 관리성을 향상시키는 아키텍처 접근 방식

<img className="jam" src={Jam} alt="JAM Stack" />

**JS** - 클라이언트의 처리는 JS가 수행

**API** - 모든 기능 및 로직은 재사용 가능한 API가 처리

**Markup** - SSG를 이용하여 Markup을 미리 생성

<img className="dif" src={Dif} alt="차이점" />

## 장점

1. 빠른 성능

**SSR**

서버가 클라이언트의 요청을 받을때마다 새로운 화면 제공

**CSR**

서버가 클라이언트의 요청을 받아 클라이언트에게 JS 파일 제공

**JAM Script (SSG)**

빌드 시점에 각 페이지의 리소스를 HTML 파일로 변환

요청이 있을 때마다 적잘한 HTML을 정적으로 제공

정적으로 제공하기 때문에 서버 리소스 사용 X

렌더링할 화면들을 미리 렌더링하여 제공되어 더 좋은 사용자 경험 제공 가능

2. SEO

SSR, CSR은 클라이언트에게 서버로부터 리소스를 다운받아 렌더링하기 전까지 빈 화면 제공

즉, 초기 HTML이 비어있기 때문에 크롤링 봇이 데이터 수집이 어려움

SSG는 빌드 시점에 정적 파일 생성하여 **미리 렌더링** 하기 때문에 크롤링에 매우 적합한 구조

3. 쉬운 자동화

**Netlify**, AWS, Azure 등을 활용하면 배포 자동화 가능

4. 큰 생태계

다양한 JAM Stack 기반 프레임워크

각 프레임워크 내 커뮤니티, 써드 파티 API

CMS, Hosting ...
