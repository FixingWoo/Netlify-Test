**Gatsby** = **React** + **JAM Stack**

## JAM Stack?

웹 애플리케이션에서 렌더링 할 화면을 **Pre-render** 하고 이를 **CDN**에서 제공하여 빠르고 안전한 앱으로 웹 서버를 관리할 필요 X

<img className="jam" src={Jam} alt="JAM Stack" />

1. JS

클라이언트의 처리(UI/UX) 수행

2. API

모든 기능 및 로직은 재사용 가능한 API가 처리
동적 또는 사용자 지정 콘텐츠 처리 (GraphQL, GA...)

3. Markup

모든 콘텐츠를 구축
마크업을 미리 빌드하고 CDN 전달

## 장점

1. 빠른 성능

**SSR**

서버에서 렌더링 준비를 마친 상태로 클라이언트에게 응답

랜더링과 동시에 JS 파일 다운로드 후 컴파일 과정 필요

**CSR**

서버 응답 결과물을 통해 클라이언트에서 렌더링 수행

**JAM Script (SSG)**

빌드 시점에 각 화면 구성 리소스를 HTML 파일로 변환

요청이 있을 때마다 리소스를 정적으로 제공

정적으로 제공하기 때문에 서버 리소스 사용 X

렌더링할 화면들을 미리 렌더링하여 제공되어 더 좋은 사용자 경험 제공 가능

2. SEO

CSR은 리소스를 다운받아 렌더링하기 전까지 빈 화면 제공

즉, 초기 HTML이 비어있기 때문에 크롤링 봇이 데이터 수집이 어려움

SSG는 빌드 시점에 정적 파일 생성하여 **미리 렌더링** 하기 때문에 크롤링에 매우 적합한 구조

3. 쉬운 자동화

**Netlify**, AWS, Azure 등을 활용하면 배포 자동화 가능

4. 큰 생태계

다양한 JAM Stack 기반 프레임워크

각 프레임워크 내 커뮤니티, 써드 파티 API

CMS, Hosting ...
