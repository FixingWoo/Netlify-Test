# 개인 블로그를 GatsbyJS로 개발한 이유

---

## Static Site Generator(SSG)란 무엇인가

SSG는 기본적으로 Markdown 또는 HTML, 데이터(JSON, YAML 또는 TOML 형식) 및 템플릿과 같은 언어를 사용하는 간단한 마크업이다.

간단히 말해 빌드 진행시 각 페이지마다 필요한 리소스를 미리 생성하고, 요청이 있을 때 정적으로 제공하는 방식을 취한다.

이렇게 정적 파일만 CDN 혹은 배포 서비스를 이용하여 배포하면 별도의 서버를 운영할 필요가 없어진다.

### 장점

1. 러닝커브

React는 JS 프레임워크이며, Gatsby는 React 기반 프레임워크이다.

그렇기 때문 자바스크립트 경험이 있다면 누구나 학습하면 Gatsby를 통해 블로그를 개발할 수 있다.

2. 다른 방식에 비해 더 빠르게 웹 사이트를 제공할 수 있다.

대부분의 웹 사이트는 처음 접속 시, 서버를 통해 데이터를 받아와 이를 렌더링하는 과정이 필요하다.

그러나 SSG는 렌더링할 화면들을 모두 Pre-Render하여 제공되어 그만큼 사용자에게 화면을 보여주기 위해 준비하는 시간을 단축할 수 있다.

마지막으로 빌드된 결과물은 모두 정적 파일이므로 CDN을 통해 쉽게 제공할 수 있다.

3. SEO

Gatsby는 React 기반의 프레임워크이다. 그리고 React는 주로 CSR 방식으로 렌더링한다.

먼저 CSR 동작 방식을 간단하게 정리해보자

서버는 요청을 받으면 클라이언트에게 JS 파일을 전달한다.

브라우저는 전달받은 JS를 다운로드하며, 동시에 유저에게 빈 화면을 보여준다.

브라우저에서 js 파일의 다운로드가 끝나면, 리액트 코드가 있는 js파일을 실행한다.

브라우저에 있는 리액트 코드가 UI를 렌더링한다.

즉, 초기 HTML 파일이 비어있기 때문에 크롤링 봇이 데이터를 수집하기 어렵게 되고 이는 SEO에 좋지 않다는 결론에 이른다.

SSG는 배포 시에 정적 파일을 생성하여 Pre-Render하기 때문에 크롤링하기 매우 적합한 구조를 가지게 됩니다.
